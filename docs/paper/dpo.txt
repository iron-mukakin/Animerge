論文『Diffusion Model Alignment Using Direct Preference Optimization』に基づき、自作アプリのsd-script（またはADDifT/NVIDIA Cosmosなどの拡散モデル学習環境）にDiffusion-DPO（Direct Preference Optimization）を実装するために必要な理論・数式、およびハイパーパラメータ等の実装要件をまとめます。
1. DPOの基本アプローチと目的関数

拡散モデルにおけるDPO（Diffusion-DPO）は、強化学習（RLHF）のように明示的な報酬モデルを学習させる必要がありません 。プロンプト（条件付け）c 、およびそれに対して生成された「好ましい画像（Win）」x0w​ と「好ましくない画像（Lose）」x0l​ のペアデータ を用い、損失関数を最小化することでモデルを直接人間の好みに整列（アライメント）させます 。  
核心となる損失関数（目的関数）

実装すべき最終的な損失関数 L(θ) は以下の通りです（論文内の式14に相当）:  
L(θ)=−E(c,x0w​,x0l​)∼D,t∼U(0,T),ϵ∼N(0,I)​[logσ(−β⋅ω(λt​)⋅Δerror​)]

ここで、Δerror​（誤差の差分）は以下のように計算されます:
  
Δerror​=(∣∣ϵw−ϵθ​(xtw​,t,c)∣∣22​−∣∣ϵw−ϵref​(xtw​,t,c)∣∣22​)−(∣∣ϵl−ϵθ​(xtl​,t,c)∣∣22​−∣∣ϵl−ϵref​(xtl​,t,c)∣∣22​)

    σ: シグモイド関数（torch.sigmoid） 。  

    β: 基準モデル（Reference）からの乖離を制御する正則化ハイパーパラメータ 。  

    ϵθ​: 学習対象（更新中）の拡散モデル（ノイズ予測器） 。  

    ϵref​: 勾配更新を行わない固定された基準モデル（Referenceモデル） 。  

    ω(λt​): タイムステップごとの重み関数。実用上は定数（通常は 1.0）として扱われます 。  

2. データ処理とフォワード（拡散）ステップ

効率的な訓練を可能にするため、逆プロセスのサンプリング軌跡をシミュレートするのではなく、通常の拡散モデルのトレーニングと同様に順方向の拡散プロセス（Forward process）を用いて近似を行います 。  

    データのサンプリング: ペアデータから、プロンプト c、好ましい画像 x0w​、好ましくない画像 x0l​ を取得します 。  

    タイムステップのサンプリング: 一様分布 t∼U(0,T) からタイムステップ t を共通（または独立）してサンプリングします 。  

    ノイズの付加: それぞれの画像に対して独立にノイズ ϵw,ϵl∼N(0,I) をサンプリングし、通常の拡散スケジュールに従って加ノイズ画像 xtw​,xtl​ を作成します 。
      
    xt∗​=αt​x0∗​+σt​ϵ∗

3. 実装に必要なコンポーネントと処理フロー

コード（sd-script内のトレーニングループなど）に組み込む際の具体的な処理の流れは以下の通りです。
① モデルの準備

    学習モデル（model）: 重みが更新されるメインのモデル。

    基準モデル（ref_model）: 初期状態（あるいはSFT後）のモデルを複製し、requires_grad_(False) および .eval() で完全に固定します。

② 損失計算の疑似ステップ

各イテレーション（ミニバッチ）における計算処理：

    学習モデルの予測値取得:

        pred_noise_w = model(x_t_w, t, c)

        pred_noise_l = model(x_t_l, t, c)

    基準モデルの予測値取得（with torch.no_grad(): 内で実行）:

        ref_noise_w = ref_model(x_t_w, t, c)

        ref_noise_l = ref_model(x_t_l, t, c)

    L2損失（MSE）の計算:

        loss_w = (epsilon_w - pred_noise_w).pow(2).mean(dim=[1,2,3])

        ref_loss_w = (epsilon_w - ref_noise_w).pow(2).mean(dim=[1,2,3])

        loss_l = (epsilon_l - pred_noise_l).pow(2).mean(dim=[1,2,3])

        ref_loss_l = (epsilon_l - ref_noise_l).pow(2).mean(dim=[1,2,3])

    DPO損失の結合:

        scale_term = -beta （※論文中では定数 T が β に内包、またはスケール調整されています ）  

        dpo_logits = scale_term * ((loss_w - ref_loss_w) - (loss_l - ref_loss_l))

        loss = -torch.log(torch.sigmoid(dpo_logits)).mean()

4. 推奨されるハイパーパラメータ設定

論文の実験セクション（SD1.5およびSDXLのファインチューニング）で示されている、実装の基準となるパラメータです 。  

    β（正則化パラメータ）:

        論文内では 2000 から 5000 の範囲が良好なパフォーマンスを示しています 。  

        SD1.5では 2000、SDXLでは 5000 が最終的に採用されています 。  

    学習率（Learning Rate）:

        DPOの勾配のノルムは β に反比例する性質があるため、学習率は β に応じて逆スケール（Inverse scaling）させることが推奨されます 。  

        論文中のSDXL（Adafactor使用時）の例： LR=β2000​×2.048⋅10−8   

    バッチサイズ:

        非常に大きな有効バッチサイズ（論文では 2048 ペア）が安定した学習に寄与しています 。リソースに応じてグラディエント・アキュムレーション（Gradient Accumulation）を多めに設定してください 。
